# Lessons

## 2026-05-08

- Tags: workflow, documentation
- Lesson: Large historical logs accumulate useful facts but are poor default working memory.
- Use: Keep durable context in short markdown summaries that can be searched quickly, then fall back to deep logs only when needed.

## 2026-05-08

- Tags: file-integrity, editing
- Lesson: This repo is unusually sensitive to partial-file edits and truncation mistakes.
- Use: Prefer targeted edits, avoid rewriting healthy large files, and verify file endings after nontrivial changes.

## 2026-05-08

- Tags: architecture, planning
- Lesson: Feature breadth is not the same as framework readiness.
- Use: When choosing priorities, favor save/load integrity, decoupling, and reusable contracts over adding more surface-area features.
