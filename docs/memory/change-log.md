# Change Log

## 2026-05-08

- Tags: tooling, workflow, memory
- Added a lightweight repo-local memory system under `docs/memory/`.
- Added canonical files for active context, architecture map, decisions, lessons, backlog, and session-level change summaries.
- Added helper scripts to search memory quickly and append timestamped notes.
- Updated `CLAUDE.md` to point future work toward the concise memory layer before using large historical logs.
- Explicitly positioned the existing Claude-agent logs as legacy deep-reference sources beneath the new concise memory layer.
- Moved bulky historical logs from the repo root into `docs/history/` and added archive `README.md`, `INDEX.md`, and `SUMMARY.md` to preserve accessibility while reducing clutter.
- Added compact memory notes derived from the archive: `archive-backlog.md`, `system-summaries.md`, and `session-checklist.md`.
- Added focused topic notes for persistence, framework-readiness, and SPD-fidelity work.
- Started closing the original-SPD identification gap with a persistent `ItemCatalog` autoload, Journal catalog wiring, and automatic potion/scroll self-identification on use.
- Added a per-run `ItemAppearance` autoload so potions, scrolls, and rings now get deterministic unidentified appearances that persist through save/load.
- Added a persistent `DiscoveryCatalog` for enemies, traps, and plants, and expanded the Journal catalog beyond items-only.
- Extended `DiscoveryCatalog` and the Journal to track weapon enchantments and armor glyphs via proc events and identified inscribed gear.
- Discovery tracking now also records revealed enemies and interacted NPCs, so the Journal catalog is not limited to kills.
- Added groundwork for talents: `TalentData` definitions, hero-owned talent points and levels, save/load support, and read-only talent display in Hero Info.
- Added a spendable talent UI with `WndTalents`, wired it into Hero Info, and added reusable window-content refresh support so talent changes update live.
- Implemented the first live talent effects: Warrior `Hearty Meal`, Mage `Empowering Meal`, Rogue `Cached Rations`, Huntress `Nature's Bounty`, and Duelist `Adventurer's Intuition`.
- Implemented the class intuition pickup hooks: Warrior `Tested Hypothesis`, Mage `Scholar's Intuition`, Rogue `Thief's Intuition`, and Huntress `Survivalist's Intuition`.
- Fixed gold item generation so random gold piles now keep their SPD `items.png` sprite index, and updated the item detail window to use the real item icon instead of a color placeholder.
- Replaced several remaining color/text-only item previews with sheet-backed icons in the shop window, alchemy window, and generic item-selection window.
- Extended that UI art cleanup to picker-style rows as well, including the shop sell picker.
- Fixed the hero combat pipeline so invisibility-based surprise attacks resolve before breaking stealth, and hero combat now routes through equipped weapon enchantment and armor glyph procs.
- Implemented the first combat talent hooks: Rogue `Sucker Punch`, Duelist `Patient Strike`, and Mage `Backup Barrier`.
- Implemented targeted thrown-item resolution through the hero action system, added a real `Shoot` action for `Spirit Bow`, and made Huntress `Followup Strike` a live ranged-to-melee combat hook.
- Hooked Huntress click-to-attack flow into `Spirit Bow`, so clicking a visible distant enemy now auto-shoots instead of always auto-walking into melee when a clear bow shot is available.
- Hardened run-reset and scene-loading loops against stale freed hero/mob references by avoiding typed `Node` iteration over arrays that can still contain invalid instances after death/restart transitions.
- Formalized the Huntress `Spirit Bow` as a dedicated equipment slot instead of a backpack special-case, and updated inventory/status/hero-info UI to reflect the separate melee and bow loadout.
- Fixed a regression from the bow-slot formalization where ranged attacks animated but exited before resolving because `throw_item` still only accepted melee-weapon or backpack ownership, not the dedicated `spirit_bow` slot.
- Routed wand zaps through the hero action system so targeted wand use now consumes a proper turn instead of resolving instantly from the item window callback.
- Added basic use-based wand identification, with zap usage progress persisted through save/load.
- Made wands properly equippable in the `misc` slot and upgraded quickslot behavior so quickslotted wands enter targeting directly for faster SPD-style tool use.
- Rebuilt the alchemy recipe layer so seed-to-potion and stone-to-scroll recipes now use valid item IDs, and updated the alchemy window to operate on real `Recipe` results instead of broken placeholder dictionaries.
- Wired `Alchemize` to actually open `WndAlchemy` anywhere through `EventBus.show_window`, and hooked the Alchemist's Toolkit into crafting so it gains progress from completed recipes.
- Implemented the missing hero `search` action by routing toolbar search through the normal hero action pipeline and calling the existing secret-door/hidden-trap reveal logic, so searching now spends a real turn and can actually discover adjacent secrets without duplicate signal handling.
- Improved search and door parity further: bumping/clicking adjacent impassable terrain now searches instead of failing silently, adjacent doors use the shared `Door` interaction logic, and keyboard `S` now triggers search directly.
- Added explicit search feedback in the main action animation path, so using search now shows a visible status/particle effect on the hero even when no secret is found.
- Restored base dewdrop generation from trampling high grass, instead of limiting dewdrops to the Huntress talent path; `Nature's Bounty` now boosts the baseline dew chance and can still add seed finds.
- Rebuilt the Sad Ghost quest flow: first interaction now starts the quest and spawns the target mob, kill tracking completes the objective, rewards use the standard quest reward window with real generated items, and the ghost now departs cleanly through the shared quest infrastructure.
- Fixed mimic visuals in both states: revealed mimics now use the original SPD `mimic.png` mob art, and disguised mimics now render using their configured fake loot item instead of a hardcoded generic potion icon.
- Removed the artificial per-visible-mob post-action delay from the turn manager so enemies still resolve in initiative order and update visually, but no longer add a noticeable pause before control returns to the player.
- Upgraded resting from a hidden one-turn wait into a real HUD action: the toolbar now has a Rest button, resting starts continuous auto-waiting, refuses to start while enemies are visible, and interrupts correctly on damage or when enemies come into view.
- Fixed secret-room entrance generation for tunneled branch connections: secret rooms attached as `neighbors` now get real `SECRET_DOOR` endpoint tiles instead of plain carved openings, so searching can reveal their entrances properly.
- Fixed the quickslot HUD/inventory flow: quickslot capacity now matches the toolbar, assignments persist in belongings serialization, the HUD refreshes from real quickslot state, and item windows now open a slot-selection sub-window instead of always assigning to slot 1.
- Replaced text-only quickslot labels in the toolbar with real `items.png` icon rendering plus slot-number overlays, fixing clipped item names and bringing quickslot visuals in line with the rest of the item UI.
- Fixed popup centering and basic viewport responsiveness: `WndBase` windows now fit within the current viewport and center themselves properly, the HUD now switches to a more compact portrait-friendly layout, and the hero-select scene no longer forces size on a fully anchored root control.
- Fixed dynamically spawned mob visibility for cases like Swarm splits: `GameScene` now ensures any alive mob present in the level but missing a sprite gets a `MobSprite` during turn refresh, preventing invisible logical enemies after split/spawn events.
- Expanded SPD sprite-sheet coverage in `MobSprite` for many valid mob and NPC IDs that already had art assets on disk, reducing cases where alerted or newly seen entities fell back to procedural placeholder art with health bars.
- Improved death presentation: generic character death now fades and settles instead of shrinking away aggressively, hero death now has a distinct collapse animation, and the death-scene transition waits long enough for that animation to read on screen.
- Closed several remaining item/content compatibility gaps at the generator boundary by normalizing old SPD-style IDs (for example `scroll_of_identify` and `potion_of_healing`) and adding proper quest-item generation for the wandmaker request items instead of falling through to generic unknown-item placeholders.
- Improved alert AI behavior: alarm/noisemaker-style alerts now propagate a disturbance position through `Level.alert_all_mobs()`, mobs enter hunting with a meaningful `target_pos`, and alerted mobs show the alert emote instead of merely flipping state silently.
- Tightened asset fidelity around mimic disguises by switching their fake loot IDs to the actual generator-supported IDs, which keeps disguise visuals aligned with the real item art path instead of relying on alias/fallback behavior.
- Continued the same cleanup pass by converting a few remaining live legacy scroll IDs at the callsite level (hero starter loadout and Unstable Spellbook effect table), so those paths no longer depend on alias translation.
- Improved mob behavior fidelity further: alerted mobs with no visible hero now investigate the disturbance location instead of immediately dropping back to wandering when `target` is null.
- Added SPD sprite-sheet coverage for Yog's fist variants so those boss entities no longer need procedural fallback art.
- Fixed animated/golden statue weapon generation by routing them through a real tiered melee-weapon helper in `Generator` instead of the invalid `melee_weapon` ID, so statue combat and reward drops now use actual enchantable weapons rather than generic fallback items.
- Fixed the shop inventory path so the shopkeeper now stocks real generated items with canonical IDs instead of placeholder `Item` objects with bad IDs like `scroll_mirror_image` and `potion_healing`, improving both purchased-item behavior and item-art fidelity in shop UI.
- Implemented the missing sheep-summon path shared by `Stone of Flock` and `Wooly Bomb`: added a real temporary `Sheep` mob blocker and wired both consumables to spawn sheep on valid open cells instead of only printing placeholder messages.
- Replaced three remaining spell placeholders with live behavior: `Recycle` now opens item selection and transmutes consumables into others of the same broad class, `Curse Infusion` now curses and upgrades a selected item, and `Summon Elemental` now creates a temporary allied fire or frost elemental that follows the hero and attacks enemies.
- Cleaned a large launch-warning batch across gameplay, UI, and utility scripts: fixed unused parameters/locals, several local-name shadowing issues, and the integer-division warnings reported from `char.gd`, `scroll.gd`, `bomb.gd`, `wand.gd`, `item_slot.gd`, `ballistica.gd`, and related files.
- Implemented two more previously placeholder runestones: `Stone of Enchantment` now applies a random weapon enchantment to the equipped melee weapon or spirit bow, and `Stone of Augmentation` now lets the player pick a weapon/armor target and choose a real augment through a new augmentation selector window.
- Fixed missing enemy attack animations by making mobs record explicit last visible action metadata (`move` vs `attack`) and having `GameScene` animate from that data instead of trying to infer attacks from the mob's post-action `target` state.
- Extended that mob-attack animation fix to cover misses too, so visible enemy melee swings now animate whether they hit or miss instead of only when `on_attack_hit()` fired.
- Replaced the immediate-detonation bomb placeholder with real fuse timing: armed bombs now live on the `Level`, tick once per hero round via `TurnManager.round_completed`, detonate before enemy turns, and persist through `RegularLevel` save/load.
- Added visible telegraphing for armed bombs: `GameScene` now renders a separate floor sprite for any bomb in `Level.pending_bombs`, applies normal fog/visited visibility rules, and shows a small `Fuse` popup when a bomb becomes active.
- Implemented the real cast path for `Unstable Spellbook`: it now instantiates the chosen scroll ID and applies that scroll's `read_scroll()` effect directly to the hero instead of only logging the chosen effect and emitting a generic event.
- Wired equipped artifact hooks into the real hero loop: `Hero.execute_action()` now calls `artifact.on_turn(self)` each hero turn, and search actions now call `artifact.on_search()` when supported, which makes artifacts like `Talisman of Foresight` actually charge and apply their passive/turn-based behaviors in play.
- Implemented a real terrain-growth effect for `Sandals of Nature`: passive charged movement can now convert `EMPTY`/`EMBERS` into `GRASS` and upgrade grass into `HIGH_GRASS`, while activation now triggers a small burst of nearby growth instead of only printing flavor text.
- Completed the basic `Sandals of Nature` charge loop in the UI: seed items now show a `Feed` action in `WndItem` when the sandals are equipped, which consumes one seed from the stack and routes it into `SandalsOfNature.feed_seed(...)`.
- Implemented the real `Master Thieves' Armband` loop: positive gold gain now routes through `on_gold_pickup()` when the armband is equipped, and activation now performs an actual adjacent-shopkeeper theft attempt that steals a real stock item on success or forces the shopkeeper to flee on failure.
- Fixed a boss-AI timing mismatch: custom `_act_hunting()` overrides for `Goo`, `Tengu`, `DM-300`, `King of Dwarves`, and `Yog-Dzewa` now explicitly spend turn/move/attack time for their special actions, restoring fair cooldown behavior under the shared turn system.
- Replaced the placeholder thief/bandit steal behavior with real inventory theft: `Crazy Thief` and `Crazy Bandit` now remove an actual backpack item (or one unit from a stack), flee after stealing it, and drop that stolen item on death instead of only toggling a boolean placeholder.
- Implemented a real first-pass `Necromancer` AI loop: it now acquires a hero target, raises one linked `Skeleton`, keeps that skeleton focused on the hero, periodically heals or hastes it, retreats when pressured, and destroys the linked skeleton when the necromancer dies.
- Fixed the same turn-cost mismatch across several non-boss ranged mobs: `Gnoll Shaman`, `Scorpio`, `Gnoll Trickster`, and `DM-100` now explicitly spend attack/move/idle time inside their custom `_act_hunting()` logic instead of performing ranged/kiting behavior without consuming proper turn cost.
- Tightened two more special-mob behaviors: `Succubus` teleport-escape now consumes the turn instead of teleporting and then also taking a full normal action, and `Stone Golem` now emits `hero_moved` when its on-hit teleport displaces the hero so the rest of the game state/UI stays in sync.
- Improved cave/prison special behavior: `Spinner` now tracks the hero's last seen position and shoots predictive webs that also seed a real `WebBlob`, instead of only dropping `WEB` terrain on the hero's current tile, and `Prison Guard` chain-pull now consumes proper attack time.
- Implemented the missing `Timekeeper's Hourglass` scheduler effect in `TurnManager`: while the hourglass freeze is active, non-hero actors are skipped by the turn loop and still advanced through cooldown accounting, so time-freeze now actually stops enemy actions instead of only tracking local artifact state.
- Wired artifact `on_floor_change()` hooks into stair transitions in `GameScene`, so per-floor artifacts like `Dried Rose` can now reset/refresh their state when moving between dungeon levels instead of carrying stale per-floor state forever.
- Hardened the save/backtrack reconstruction path: `LevelFactory` now supports subclass instantiation without generation, `SaveManager` recreates the correct level subclass on cold load, `Level` now serializes/deserializes mobs/traps/plants/blobs/armed bombs directly, base `Mob` deserialization now restores core combat/AI state instead of being a stub, and cached-level loading in `LoadingScene` no longer stacks fresh quest/shop spawns on top of restored level state.
- Fixed Sad Ghost/NPC interaction regressions: adjacent NPC clicks in `GameScene` now route to `interact` instead of the generic adjacent-attack path, and `SadGhost` now uses its own wandering turn loop/state handling instead of inheriting the passive NPC act path that left it stationary and non-behaving.
- Fixed a death-phase turn-loop bug: `TurnManager` now aborts the async enemy-processing loop if the hero dies or the `GameScene` has already ended, and `GameScene._on_hero_died()` now explicitly drops turn-manager processing flags so the floor cannot keep resolving visible enemy turns during the death-scene delay.
- Continued save hardening: `SaveManager` no longer restores blobs twice after the level serializer absorbed them, hero/mob active buff state now persists through save/load using script-path-based buff reconstruction, and `Necromancer` now relinks its summoned skeleton after level load using saved `actor_id` references instead of dead runtime instance IDs.
- Closed a follow-on persistence hazard in actor identity: deserializing an `Actor` now bumps the global `_next_id` floor to at least `actor_id + 1`, preventing newly spawned mobs/allies/projected actors after load from colliding with restored actor IDs.
- Extended persistence into the quest/NPC layer: base `NPC` save/load now restores quest flags and re-registers restored quest NPCs with `QuestHandler`, and custom state for `Sad Ghost`, `Wandmaker`, `Blacksmith`, and `Shopkeeper` now persists instead of falling back to generic mob-only restoration.
- Repaired the Imp quest path: `Ambitious Imp` now has a coherent single-ring reward flow with real generation, reward handoff, departure, and save/load state, instead of referencing undefined reward fields/functions; also fixed `QuestHandler` to call `Wandmaker.create_quest_item(...)` instead of the non-existent `create_quest_seed(...)` when spawning the wandmaker quest item.
- Continued the quest/NPC runtime cleanup: `Wandmaker` now generates real wand rewards with canonical IDs, uses correct inventory-ID/quantity checks for handing over quest items, and only completes the quest when a reward is actually chosen; `WndQuestReward` now emits an explicit reward-selection signal so quest NPCs no longer incorrectly depart just because the window closed. `Blacksmith` now uses a real `WndReforge` flow, consumes ore through quantity-safe inventory removal, and applies the actual reforge rule/cleanup through the NPC logic instead of leaving reforging as a dead text-only branch.
- Added a real root `README.md` for the project, documenting what the port is, current `0.1.1` status, how to run it in Godot 4.5, the repository layout, current goals, and the main known rough edges.
