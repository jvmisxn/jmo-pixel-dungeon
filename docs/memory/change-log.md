# Change Log

## 2026-05-09

- Tags: input, ui, qol
- Added explicit keyboard attack binds in `GameScene`: `A` and `F` now attack the first adjacent hostile target, with a warning if no adjacent enemy is available.
- Enlarged the popup map window again by letting `WndMap` use a much larger viewport fraction and reducing frame padding, so the opened map takes up more of the screen instead of remaining close to minimap size.
- Continued the fallback-item cleanup by fixing the Imp reward pool to use canonical `ring_of_sharpshooting` instead of the bad `ring_sharpshoot` identifier, and added generator aliases so older save data or stray legacy callsites still resolve to the real ring instead of a generic fallback item.
- Continued that cleanup in the Wandmaker quest by switching the rotberry request/item path from legacy `rotberry_seed` to canonical `seed_of_rotberry`, so the quest now uses the real seed ID end-to-end instead of depending on alias fallback behavior.
- Audited grass/level-generation feel and adjusted three high-signal mismatches: trampled grass now drops a real visible `Dewdrop` heap instead of instant auto-healing, standard rooms no longer spray large amounts of grass into every normal floor, and `Feeling.GRASS` now paints a more controlled mix of grass/high grass instead of turning whole standard-room interiors into blanket grass tiles.
- Continued the grass-system cleanup by replacing generic placeholder seed items with a real `SeedItem` class: generated seeds now support actual targeted planting, the item window shows a `Plant` action for seeds, and throwing seeds through the normal hero action path now plants/activates them correctly instead of treating them like inert generic items.
- Tightened the seed/plant flow further by making Wandmaker's rotberry quest item a real `SeedItem` instead of a generic item, and by removing the duplicate post-activation plant erase in `GameScene` so plant cleanup is owned consistently by `Plant.activate()`.
- Fixed a real Sad Ghost/Imp quest-tracking bug by extending `mob_defeated` to carry both `mob_name` and stable `mob_id`, then switching quest kill matching to use `mob_id` where appropriate. This fixes cases like `Gnoll Trickster` where the display name did not match the stored quest target identifier.
- Added a real plant render path: `GameScene` now tracks planted flora separately from terrain and items, and new `PlantSprite` nodes give planted seeds/plants a distinct on-map presentation instead of reading only as a `HIGH_GRASS` terrain change.
- Added plant placement/activation feedback on the effect layer: planting now shows a small burst and `Planted` status, and triggering major plants now emits plant-specific color/status VFX like `Regen`, `Armor`, `Burn!`, `Frozen`, `Poison`, `Rooted`, `Sleep`, `Warp`, `XP`, and `Haste`.
- Tightened three thinner plant mechanics: `Stormvine` now applies both `Rooted` and `Vertigo`, `Fadeleaf` now avoids obviously bad teleport destinations and emits `hero_moved` for hero teleports, and `Dreamfoil` now explicitly puts mobs back into the `SLEEPING` AI state instead of using a raw integer state write.
