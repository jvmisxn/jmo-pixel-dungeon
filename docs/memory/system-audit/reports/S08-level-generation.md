# Level Generation — Audit

- Files: `src/levels/builders/{builder,loop_builder}.gd`, `src/levels/painters/{painter,standard_painter}.gd`, `src/levels/rooms/**` (room base + entrance/exit/connection/standard + special/* + secret/*). Orchestrated by `src/levels/regular_level.gd`.
- Read in full: yes (builders, painters, room base + standard/connection/entrance/exit + representative special/secret rooms; remaining special/secret rooms are structural clones of the ones read).
- Verdict: **needs-hardening** — geometry/painting core is clean and SPD-shaped, but the door/connection layer is effectively dead: the builder always leaves a 1-tile gap, so `connect_adjacent()` never succeeds and `Room.connected` stays empty for every regular-level room. All doors (including vault/armory **LOCKED_DOOR** and shop doors) are therefore never painted; rooms join only via carved open tunnels.

## Improvements
- [P1] **`connected`/door system is dead on regular levels.** `Builder.place_adjacent()` positions rooms with `gap = 2` (`_position_on_side`, builder.gd:44) and `_is_valid_placement` rejects anything within `margin=1`, so placed rooms are always ≥1 empty tile apart. `Room.find_door_pos()` (room.gd:155) only returns a cell when `right == other.left` (walls touching), which now never happens → `connect_adjacent()` (builder.gd:101) always returns false → `connect_to()` is never called → `connected` is empty for all rooms. Consequence: `paint()` in standard/entrance/exit/shop/**vault**/**armory**/garden/etc. never places DOOR/LOCKED_DOOR (they iterate an empty `connected`), and `GardenRoom`'s door→center grass path is never drawn. **Gameplay impact:** vaults/armories have no locked door (free treasure, key gate bypassed); every room is an open tunnel mouth. SPD keeps adjacent rooms wall-sharing with doors and reserves tunnels for non-adjacent pairs. Direction: allow zero-gap adjacency (share a wall) so `find_door_pos` can succeed, or place a DOOR/LOCKED_DOOR tile at the tunnel mouth for `neighbor`-linked rooms in `_carve_tunnels`. Verify in-engine (LOCKED_DOOR present on a depth-10+ vault floor). (builder.gd:36-97, room.gd:155-177, standard_painter.gd:35-71, vault_room.gd:36, armory_room.gd:42)
- [P2] **Tunnel `pair_key` can overflow/collide.** `mini(id) * 100000 + maxi(id)` (standard_painter.gd:49) uses `get_instance_id()`, which for RefCounted objects has the high bit set (very large / negative-signed 64-bit values). `*100000` overflows int64 and wraps, so distinct room pairs can hash to the same key and a tunnel gets silently skipped — a potential disconnected floor (currently masked by the 15× regen + `find_path` traversability check in regular_level.gd:74-77). Direction: key on a `Vector2i(min_id, max_id)` or a `"%d_%d"` string.
- [P2] **`_place_connection_rooms` is one-sided.** It shares a single `conn_idx`, only calls `place_adjacent(conn, a, …)` (never tries side `b`), and marks conn↔b as neighbors regardless of geometry (loop_builder.gd:216-235). Connection rooms can float far from `b`, producing long L-tunnels instead of a real junction. Direction: attempt placement between the midpoint of a/b, or fall back to skipping the conn room rather than force a distant neighbor link.

## Optimizations
- [P2] **`_generate_patch` runs full-map cellular automata per water level.** For every Sewer/Caves/City level it allocates a `WIDTH*HEIGHT` bool array and re-allocates it `smoothness` (4–6) times over the whole grid (standard_painter.gd:203-233), then samples only room interiors. Restrict the automaton to the union of room-interior cells (or cache/threshold) to cut the per-floor allocation + O(smoothness·LEN) work.
- [P3] **`_decorate_walls` scans all `LEN` cells with a 4-neighbor probe** (standard_painter.gd:130-144). Fine, but could iterate only wall cells adjacent to painted room borders collected during painting.

They were clearly meant to gate tunnel-mouth door placement — wire them into the P1 fix or delete. *(File is in TRUNCATED_FILES.txt — do not auto-edit.)*
- [P3] **Dead/parity-orphan methods in `room.gd`:** `random_door_pos()` (line 79) and `on_border()` (line 67) have zero call sites. They belong to the door system that the P1 gap bug left unwired; keep them as the hook for the door fix rather than deleting.
- [P3] **Only `LoopBuilder` exists.** SPD ships several builders (Line, FigureEight, Regular) chosen per depth for layout variety. All regular floors here use one loop shape. Add alternate builders behind a per-region/per-depth selector.
- [P3] **`_water_params` applies 30% water fill to every Sewer/Caves/City floor regardless of feeling** (standard_painter.gd:173-189), not just WATER-feeling floors. Confirm against SPD region defaults; may over-water non-water floors.

## Save/load & coupling notes
- Builders/painters/rooms are transient generation-time objects (`RefCounted`), not serialized — the level persists only the final `map`/terrain arrays (via `Level.serialize`, S01/S07). No save/load risk in this system itself, but the door bug means saved floors permanently lack locked doors.
- Coupling: painters/rooms reach into `Level` fields directly (`level.map`, `level.rooms`, `level.entrance`, `level.set_terrain`) and depend on `ConstantsData` (WIDTH/HEIGHT/Terrain/DIRS_4) and region classes (`SewerLevel`/`CavesLevel`/`CityLevel` via `is` checks in `_water_params`) — a region-type switch inside the painter is a mild layering inversion.

## Research notes
- Reasoned against Shattered PD's `Builder`/`RegularBuilder` + `Room.Door` model: SPD places neighboring rooms wall-adjacent and installs `Door` objects on shared walls (regular/empty/hidden/barricade/locked), carving tunnels only for non-adjacent graph edges. This port collapsed everything to gapped tunnels, which is why the `connected` door layer never fires — the root of the P1 finding.
- SPD `SpecialRoom.maxConnections()==1`, `ShopRoom==2`, `SecretRoom==1` are faithfully mirrored (garden_room.gd:10, shop_room.gd:12, secret_room.gd:11), but the limits are currently moot because no `connected` doors are ever created.
