# Session Checklist

## Before Work

1. Read `active-context.md`.
2. Search `docs/memory/` for the system or feature name.
3. Read `archive-backlog.md` if the task sounds like an old deferred issue.
4. Search `docs/history/` only if the concise notes are insufficient.

## During Work

1. Watch for save/load implications.
2. Watch for autoload coupling and hidden side effects.
3. Avoid widening scope unless the archive already shows the systems are tightly linked.

## After Work

1. Add a short entry to `change-log.md`.
2. Update `active-context.md` if priorities changed.
3. Add to `decisions.md` only when future choices should be constrained.
4. Add to `lessons.md` only when it will prevent repeated mistakes.
